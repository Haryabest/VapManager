# Project: Qt (C++/H) → QML Migration

## Goal
Полностью переписать UI на QML с минимальными затратами токенов и без лишних объяснений.

---

## General Rules
- Отвечай **только кодом** (без пояснений, комментариев вне кода и "воды").
- Не повторяй код, если он уже был показан ранее.
- Пиши **минимально необходимый код**, без избыточных обёрток.
- Используй **краткие имена переменных**, если это не ухудшает читаемость.
- Не генерируй лишние файлы — только те, что реально нужны.

---

## File Handling
- Читай **только релевантные файлы**, не загружай весь проект.
- Не дублируй содержимое файлов без необходимости.
- Если файл не меняется — не выводи его.

---

## QML Guidelines
- Используй **чистый QML без избыточного JS**.
- Предпочитай:
  - `anchors` вместо сложных layout'ов
  - `property` вместо лишних функций
- Минимизируй вложенность компонентов.
- Выноси переиспользуемые элементы в отдельные QML-компоненты **только при необходимости**.

---

## C++ Integration
- Оставляй C++ только для:
  - бизнес-логики
  - работы с системой
- Используй `Q_PROPERTY` и `Q_INVOKABLE` вместо сложных биндингов.
- Убирай UI-логику из C++.

---

## Token Optimization
- Не объясняй очевидные вещи.
- Не добавляй комментарии, если их не просят.
- Не форматируй код избыточно.
- Используй короткие конструкции (тернарники, inline биндинги).
- Не расписывай альтернативные варианты — только один лучший.

---

## Output Format
- Только код
- Без markdown-описаний
- Без лишних переносов
